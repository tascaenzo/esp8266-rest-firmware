ESP8266 GPIO Automation Firmware

Secure, deterministic REST firmware for ESP8266-based devices.

This project provides a lightweight and deterministic automation firmware designed specifically for ESP8266-based boards such as ESP-01, ESP-12, NodeMCU, and Wemos D1 Mini.

The firmware exposes a JSON-based REST API that allows external clients to configure GPIOs, schedule actions, inspect device state, and manage system-level settings.
Special care has been taken to ensure that all operations remain suitable for low-memory embedded environments, avoiding dynamic allocation patterns and unpredictable execution paths.

Design Goals

The firmware is built around a small set of strict design principles. These constraints guide both the internal architecture and the exposed API surface.

  • Predictable and deterministic runtime behavior
  • Bounded memory usage with no unbounded heap growth
  • Stateless REST API suitable for automation systems
  • Replay-safe authentication without TLS overhead
  • Persistent configuration with automatic restore on reboot

Security Model

To avoid the complexity and memory overhead of TLS, the firmware implements an optional authentication layer based on a nonce-driven HMAC mechanism.

Each authenticated request is signed using a one-time nonce and a shared secret stored securely on the device. Nonces are single-use, time-limited, and bound to the client IP address, making replay attacks infeasible even on unencrypted networks.

What You Can Do in This Console

This web console is designed as a companion tool for developers and system integrators working with the firmware. It is not intended as a generic end-user interface, but as a technical inspection and experimentation environment.

  • Inspect the live state of the device and all GPIOs
  • Explore authenticated and unauthenticated API behavior
  • Study request signing and payload rules
  • Debug GPIO modes, safety levels, and cron execution
  • Configure system-level settings and authentication
